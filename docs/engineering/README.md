# STOP.

Before touching any code:

**Read every document inside `/docs/engineering/`.**

Do not begin implementation until the architecture, invariants, workflow, ADRs, lessons learned, and verification requirements are fully understood.

Every task must follow these documents.

Never optimize only for the requested feature. Always optimize for the long-term health of the entire platform.

If a requested implementation violates these principles, stop, explain why, propose alternatives, and only proceed once the design is consistent with the engineering constitution.
